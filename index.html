What's Happening

  • Date: January 2017
  • Location: UofT Hacks, Toronto, Canada
  • Technologies used: Angular 2, Ionic, Cordova, HTML, CSS, Firebase

What's Happening is a mobile app for discovering the latest nearby events.


Features

  • Users upload their events (Happenings) to What's Happening
  • Other users search for Happenings near them, filtering by various categories, such as price, duration, and more!
  • What's Happening provides directions, allowing users to go and have a great time.

How it works

The backend is supported by the Firebase platform. Each Happening is given a unique hash, then stored under a single node in the firebase system. Users hosting Happenings upload key information about the Happening, such as the name, a description, address, and more. The UI is created with HTML and CSS, with functionality implemented with JavaScript. Ionic is used in conjunction with Cordova in order to port the project to support both Android and iOS devices.


What makes it different

One might wonder, with Facebook events and Eventbrite already around, do people really need What's Happening. What's Happening provide a quick and easy way to advertise your event. There is no need to sign up for an account. You simply have to create an event and create a password (in case you want to cancel your event). Furthermore, it's great for small-local activities, for example, if you want to find people who's also interested in playing basketball this afternoon.

Dynamic User Driving Enhancement (DUDE)

  • Date: June 2017
  • Location: General Motors Hack AI, Toronto, Canada
  • Technologies used: NGI SDK, JavaScript, HTML, CSS, Google Maps API
  • GM Hack AI: First Place

DUDE is a personalized assistant for level 3 autonomous driving cars with functionalities tailored towards different demographics. Vehicle users can create individual accounts to store their tailored preferences that toggle specific functions.


Features

  • When the Speed Warning function is enabled, the car will send a text message to a specified phone number if the car is exceeding the current road's speed limit as dictated by Google Maps.
  • When the Location Monitoring option is selected, the car will text its geographical position to a specified phone number over intervals of time.
  • The Enhanced Accessibility option alters the user interface to better fit the visual needs of the elderly or those with special vision needs.
  • Learner Mode provides safety assistance to beginning drivers such as speed monitoring, seat belt reminders, and blinker usage, which is done through keeping track of the various turning and signal behaviours of the car.

Education Networking Tree (ENT)

  • Date: September 2017
  • Location: RBC Next Great Innovator Hackathon, Toronto, Canada
  • Technologies used: Angular 2, Ionic, Cordova, HTML, CSS, Firebase

The hallmark of an exciting and worthwhile career is the chance to constantly challenge oneself with new concepts. While researching for this hackathon, we talked to several experienced software developers and asked them what they thought was the least desirable situation that can occur in the workplace. The general consensus was that employees dislike not learning anything new, and doing the same work day in and day out--like a machine. Therefore, we decided to build an app around the theme, “The Future of Work”, that will avoid this pitfall and keep RBC an attractive employer to the new generation.

ENT is an internal, cross-platform application for RBC that will be made available to all employees. Our app allows employees working in different departments to connect with each other and learn more about different disciplines. Our goal is to facilitate knowledge and idea exchanges in the workspace, be it among interns, new hires, or seasoned full-timers. As a result, we will help build an actively engaged community at RBC.

For each skill area, the user can view a list of learning opportunities related to that particular area in the Learn section. Events can include anything from seminars to workshops. In the Ask section, they will be able to ask questions and/or connect with a mentor. Additionally, ENT provides a Social aspect to promote community activities.

CAPTCHA Solver

  • Date: July 2017
  • Location: Hack MIT Puzzles
  • Technologies used: Python, Keras, TensorFlow

The goal for the puzzle is to solve 10,000 out of 15,000 CAPTCHAs successfully. The lines were first removed from around 1000 CAPTCHAs and the pictures were then turned to black and white. After, each CAPTCHA was separated into four images, one letter or number per image, depending on where the characters were. Afterwards, the characters were fed to the neural network as training data.
